## Findings considered and rejected

- **`PdfCanvas.tsx` decomposition (920 lines, ~10 concerns)**: real debt, but L effort with the highest regression surface in the app; postponed until a feature next forces a change there.
- **PageRail thumbnail virtualization**: valid at 100+ pages; deferred — typical documents are small, and `React.memo` already removed the repeated re-render cost.
- **Persisting undo history (`past`/`future`) to IndexedDB on autosave**: needs a product decision on whether undo must survive reload; not a pure code fix.
- **`react-select` replacement for the font picker**: bundle-size claim unmeasured; run `vite build` with a bundle analyzer before spending the effort.
- **78-file prettier reformat + `format:check` CI gate**: correct but belongs in its own dedicated PR to avoid drowning functional diffs.
- **`bun update jsdom` (undici advisories)**: dev-only exposure, zero production path; clear opportunistically.
- **React 18→19**: no cost to staying on 18 today; bundle with a future react-pdf/pdfjs-dist bump.
- **Multi-select alignment commands / CSV bulk form-fill / PNG export**: grounded direction ideas, not defects; PNG claim removed from README instead.
